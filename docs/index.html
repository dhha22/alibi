Page 02 · The Investigation

What happens when you ask

You ask “There's a mystery setTimeout(resolve, 0) in db/client.ts:88 — can I delete it?”
  1. 1

    Blame — with the right options

    alibi doesn't run bare git blame. It adds -w and the repo's ignore-revs file, because the bare form easily points at the wrong commit:

    $ git blame -w -L 88,88 db/client.ts a1b2c3d (@dohyun 2023) "style: apply prettier" — 412 files changed
  2. 2

    The noise-judgment loop

    Is that commit formatting noise? alibi proves it (empty -w diff, mass-touch shape), excludes it, and re-blames — repeating until it reaches the commit that authored the line's meaning:

    $ git blame -w --ignore-rev a1b2c3d -L 88,88 db/client.ts 4f2a91c (@minji 2021) "fix: safari indexeddb tx race" ← the true origin
  3. 3

    Chase the story off-git

    With the gh CLI it follows the origin commit into the PR, linked issues, and review comments — where the actual reason usually lives. (No gh? It concludes on git history alone and says so.)

    $ gh api repos/:owner/:repo/commits/4f2a91c/pulls PR #482 — "Safari 14 WebKit bug workaround" reviewer: "ugly but necessary — remove when WebKit fixes it"
  4. 4

    Check whether the reason still holds today

    The verdict hinges on now, not 2021: does the bug still exist? Is the platform still supported? Here BROWSERS.md shows Safari 14 support was dropped in 2023 — the reason no longer holds. Verdict: ⌛ EXPIRED.

Page 03 · The Case File

What you get back

Every investigation ends in this report — exhibits (E-n), witness statements (W-n), a deduction chain, and a verdict. No evidence, no claim. The final call is yours.

🗂️  CASE #4f2a91c — "can this setTimeout(resolve, 0) be deleted?"
────────────────────────────────────────────────────────

⌛ Verdict: EXPIRED — the Safari bug it guarded against died in 2023.

📋 Case summary
   Introduced 2021 by PR #482 (@minji) to dodge a Safari 14 IndexedDB
   transaction race. blame initially framed @dohyun's prettier sweep;
   the real author is @minji.

🧾 Exhibits
   E-1  commit 4f2a91c "fix: safari indexeddb tx race" (2021, @minji)
   E-2  PR #482 — links the WebKit bug, explains the 0-delay yield
   E-3  BROWSERS.md — Safari 14 support dropped 2023-06

🗣️  Witness statements
   W-1  @reviewer (2021): "ugly but necessary — remove when WebKit fixes it"

∴  Deduction
   E-2 (Safari-14-only bug)      ∴ the reason was real
   E-3 (Safari 14 unsupported)   ∴ the reason no longer applies
   ∴  Verdict: EXPIRED

⚖️  Disposition recommendation (the decision is yours)
   Safe to remove. Keep E-2's repro scenario as a regression test.

🔗 Related   SHA: 4f2a91c   PR: #482
⚖️ JUSTIFIED

The reason was real — and still holds.

→ Don't touch it. It exists for a reason.
⌛ EXPIRED

The reason was real, but it's gone now.

→ Safe to remove, precautions attached.
🕳️ COLD CASE

The reason could not be found. alibi never invents one.

→ Maximum caution. Here are the last witnesses.
Page 04 · Six Cases

Six kinds of questions it answers

Ask naturally — alibi picks the investigation mode from the shape of your question. Each mode returns a different report, always evidence-pinned, always with a verdict or disposition.

why-not-whodefault
“Why is this timeout 30 seconds? Can I change it?”
A case file (like the one above): true origin commit, PR rationale, verdict on whether the reason still holds.
commit-to-story
“Give me the full story of commit 4f2a91c — why did this land?”
A dossier: the problem it solved → the debate in review → the decision → what happened after it shipped.
first-broken
“Export worked last week and now returns 0 — find the commit that broke it.”
A stakeout log: repro probe validated on both ends, git bisect run, the culprit commit interrogated — accident or intent.
incident-trace
“Was the parallel-upload change ever reverted? Which releases had it?”
An incident timeline: land → revert → reland chain, plus an Exposure line — exactly which tags/releases carried the bug.
design-rationale
“Why is upload a sequential queue and not Promise.all? Was anything else tried?”
A precedent record: the design eras, the alternatives tried and rejected (with proof), and whether the decision still holds today.
repo-tour
“I just joined — onboard me through this repo's history. Where are the landmines?”
A five-scene repo tour: the churn hotspots, repeatedly reverted areas, load-bearing old code you shouldn't touch, blame hygiene — and your safest first task.
Page 05 · Install

Install

Two prompts inside Claude Code and you're set. With the gh CLI authenticated it also reads PR and review threads; without it, it works from git history alone.

# install as a Claude Code plugin
/plugin marketplace add dhha22/alibi
/plugin install alibi@alibi

# or install the skill by hand
git clone https://github.com/dhha22/alibi
cp -r alibi/skills/alibi ~/.claude/skills/alibi

Connect the gh CLI (optional) — to pull in PRs, issues, and review threads as evidence, install the gh CLI and authenticate once. Without it, alibi still works from git history alone.

# install the gh CLI (macOS · Homebrew)
brew install gh

# authenticate your GitHub account — one browser prompt
gh auth login

# verify the connection
gh auth status
No code is guilty. It just has a past.
