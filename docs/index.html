FILE 01What happens when you ask

You ask “There's a mystery setTimeout(resolve, 0) in db/client.ts:88 — can I delete it?”
  1. ①

    Blame — properly armed

    alibi never runs bare git blame. It arms it with -w and the repo's ignore-revs file, because the bare form is precisely the tool that lies:

    $ git blame -w -L 88,88 db/client.ts a1b2c3d (@dohyun 2023) "style: apply prettier" — 412 files changed
  2. ②

    The noise-judgment loop

    Is that commit formatting noise? alibi proves it (empty -w diff, mass-touch shape), excludes it, and re-blames — repeating until it reaches the commit that authored the line's meaning:

    $ git blame -w --ignore-rev a1b2c3d -L 88,88 db/client.ts 4f2a91c (@minji 2021) "fix: safari indexeddb tx race" ← the true origin
  3. ③

    Chase the story off-git

    With the gh CLI it follows the origin commit into the PR, linked issues, and review comments — where the actual reasons live. (No gh? It closes the case on git evidence alone and says so honestly.)

    $ gh api repos/:owner/:repo/commits/4f2a91c/pulls PR #482 — "Safari 14 WebKit bug workaround" reviewer: "ugly but necessary — remove when WebKit fixes it"
  4. ④

    Check whether the reason still holds today

    The verdict hinges on now, not 2021: does the bug still exist? Is the platform still supported? Here BROWSERS.md shows Safari 14 support was dropped in 2023 — the reason is dead. Verdict: ⌛ EXPIRED.

FILE 03Six kinds of questions it answers

Ask naturally — alibi picks the investigation mode from the shape of your question. Each mode returns a different report, always evidence-pinned, always with a verdict or disposition.

why-not-whodefault
“Why is this timeout 30 seconds? Can I change it?”
A case file (like the one above): true origin commit, PR rationale, verdict on whether the reason still holds.
commit-to-story
“Give me the full story of commit 4f2a91c — why did this land?”
A dossier: the problem it solved → the debate in review → the decision → what happened after it shipped.
first-broken
“Export worked last week and now returns 0 — find the commit that broke it.”
A stakeout log: repro probe validated on both ends, git bisect run, the culprit commit interrogated — accident or intent.
incident-trace
“Was the parallel-upload change ever reverted? Which releases had it?”
An incident timeline: land → revert → reland chain, plus an Exposure line — exactly which tags/releases carried the bug.
design-rationale
“Why is upload a sequential queue and not Promise.all? Was anything else tried?”
A precedent record: the design eras, the alternatives that were tried and rejected (with proof), and whether the decision still holds today.
repo-tour
“I just joined — onboard me through this repo's history. Where are the bodies buried?”
A five-scene precinct tour: the churn hotspot, the scar (revert saga), the load-bearing old guard, blame hygiene — and your safest first move.

FILE 04Who's on the case

personnel file — homicide div.

A homicide detective — 20 years on the force, arrest rate zero, on purpose. As a rookie, a botched rebase left his name on a hotfix he never wrote, and he alone was grilled for it. Since that day he doesn't trust the name blame points at: he asks “what happened here”, never “who did this”. He has no authority to approve or reject anything — he establishes the facts; the decision is always yours.
