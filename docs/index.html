alibi.

git blame tells you who. alibi tells you why.

A homicide detective, 20 years on the force, arrest rate zero. Point him at a suspicious line — he skips the prettier commits blame keeps framing people with, digs down to the commit that actually authored the line's meaning, chases it through PRs, issues, and review threads, and rules on its alibi.

As a rookie, a senior's botched rebase left his name on a midnight hotfix he never wrote — and he alone was grilled for it at the incident review. Since that day he does not trust the name blame points at. When he sees strange code he never asks “who did this” — he asks “what happened here.”

FILE 02blame frames the wrong witness

Run bare git blame on any line that's lived a few years and it names whoever last ran prettier over the file. The real author — and the real reason — is buried under formatting sweeps, lint fixes, renames, and file moves. alibi runs a noise-judgment loop: judge the commit blame names, prove it's mechanical, exclude it, re-blame, repeat — until it reaches the commit that authored the line's meaning.

✕ What blame says

$ git blame db/client.ts -L 88,88 a1b2c3d (@dohyun 2023) setTimeout(resolve, 0) → "style: apply prettier" — 412 files changed. @dohyun never wrote this line.

The name on the report belongs to a formatting commit. Ask @dohyun why the line exists and you'll both waste an afternoon.

○ What alibi finds

$ blame -w --ignore-rev a1b2c3d … 4f2a91c (@minji 2021) setTimeout(resolve, 0) → "fix: safari indexeddb tx race" · PR #482 reviewer: "remove when WebKit fixes it"

Origin commit found, PR chased, the reviewer's promise recovered — now the question “can I delete this?” has an answerable shape.

FILE 03Every case ends in one of three verdicts

This is a Chesterton's Fence reader. The question is always some form of “this code looks wrong — may I remove it?” — and the answer never hides in hedging. Every claim in the report cites an exhibit (E-n) or a witness statement (W-n). No evidence, no claim.

⚖️

JUSTIFIED

The reason was real — and still holds today. The bug it works around still exists; the platform it serves is still supported.

→ Don't touch it. The fence has a purpose.
⌛

EXPIRED

The reason was real, but it died — the bug was fixed upstream, the platform was dropped, the migration finished.

→ Safe to remove, precautions attached.
🕳️

COLD CASE

The reason could not be found — lost records, a verbal decision, history squashed on import. alibi never invents a plausible story to close a case.

→ Maximum caution. Here are the last witnesses.
🕊️ People are never suspects. Every human in a case file is a witness — never a blame target. The entire point of this skill is escaping blame culture: “@minji is not a suspect; she is this case's key witness.”

FILE 05What a closed case looks like

The mystery setTimeout(resolve, 0) from FILE 02, closed. Exhibits, witness statements, a deduction chain, a verdict — and the disposition stays yours.

🗂️  CASE #4f2a91c — "can this setTimeout(resolve, 0) be deleted?"
────────────────────────────────────────────────────────

⌛ Verdict: EXPIRED — the Safari bug it guarded against died in 2023.

📋 Case summary
   Introduced 2021 by PR #482 (@minji) to dodge a Safari 14 IndexedDB
   transaction race. blame initially framed @dohyun's prettier sweep;
   the real author is @minji.

🧾 Exhibits
   E-1  commit 4f2a91c "fix: safari indexeddb tx race" (2021, @minji)
   E-2  PR #482 — links the WebKit bug, explains the 0-delay yield
   E-3  BROWSERS.md — Safari 14 support dropped 2023-06

🗣️  Witness statements
   W-1  @reviewer (2021): "ugly but necessary — remove when WebKit fixes it"

∴  Deduction
   E-2 (Safari-14-only bug)      ∴ the reason was real
   E-3 (Safari 14 unsupported)   ∴ the reason no longer applies
   ∴  Verdict: EXPIRED

⚖️  Disposition recommendation (the decision is yours)
   Safe to remove. Keep E-2's repro scenario as a regression test.

FILE 07Install

One clone into your Claude Code skills directory. The detective reports for duty on the next suspicious line. Works best with the gh CLI authenticated (for PR and review-thread witness statements) — but degrades gracefully to git-only evidence without it.
