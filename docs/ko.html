alibi.

git blame은 누가를 말한다. alibi는 왜를 말한다.

Claude Code에 “이 줄 왜 있어요? 지워도 돼요?”라고 물으면 — alibi가 blame이 엉뚱한 사람에게 뒤집어씌우는 포매팅 커밋들을 걷어내고, 그 줄을 진짜로 쓴 커밋을 찾아, PR과 리뷰 스레드까지 읽은 뒤 판결이 붙은 사건 파일로 돌려줍니다.

판결 읽기

모든 답은 세 가지 판결 중 하나와 그 근거로 끝난다 — 그래서 다음에 뭘 해야 할지가 바로 나온다:

⚖️ JUSTIFIED → 건드리지 마라 ⌛ EXPIRED → 지워도 안전 🕳️ COLD CASE → 이 사람들에게 물어봐라

FILE 01물으면 실제로 일어나는 일

당신의 질문 “db/client.ts:88에 정체불명의 setTimeout(resolve, 0)이 있는데 — 지워도 되나요?”
  1. ①

    blame — 제대로 무장하고

    alibi는 맨 git blame을 절대 돌리지 않는다. -w와 리포의 ignore-revs 파일로 무장한다 — 맨 blame이야말로 거짓말하는 도구이기 때문에:

    $ git blame -w -L 88,88 db/client.ts a1b2c3d (@dohyun 2023) "style: apply prettier" — 412개 파일 변경
  2. ②

    노이즈 판별 루프

    저 커밋은 포매팅 노이즈인가? alibi가 입증하고(-w diff 비어 있음, 대량 파일 변경), 제외하고, 다시 blame — 그 줄의 의미를 쓴 커밋에 닿을 때까지 반복한다:

    $ git blame -w --ignore-rev a1b2c3d -L 88,88 db/client.ts 4f2a91c (@minji 2021) "fix: safari indexeddb tx race" ← 진짜 기원
  3. ③

    git 바깥의 이야기 추적

    gh CLI가 있으면 기원 커밋을 PR, 연결된 이슈, 리뷰 코멘트까지 따라간다 — 진짜 이유가 사는 곳이다. (gh가 없으면? git 물증만으로 사건을 종결하고, 그 사실을 정직하게 밝힌다.)

    $ gh api repos/:owner/:repo/commits/4f2a91c/pulls PR #482 — "Safari 14 WebKit 버그 우회" 리뷰어: "추하지만 필요함 — WebKit이 고치면 제거할 것"
  4. ④

    그 이유가 오늘도 유효한지 검증

    판결의 기준은 2021년이 아니라 지금이다: 그 버그가 아직 있나? 그 플랫폼이 아직 지원되나? 이 사건에선 BROWSERS.md가 Safari 14 지원이 2023년에 끊겼음을 보여준다 — 이유는 죽었다. 판결: ⌛ EXPIRED.

FILE 02돌아오는 결과물

모든 수사는 정확히 이 보고서로 끝난다 — 증거물(E-n), 증인 진술(W-n), 추론 체인, 판결. 증거 없으면 주장 없고, 판결 없으면 보고서도 없다. 처분은 끝까지 당신의 몫이다.

🗂️  CASE #4f2a91c — "이 setTimeout(resolve, 0) 지워도 되나?"
────────────────────────────────────────────────────────

⌛ Verdict: EXPIRED — 이 줄이 막던 Safari 버그는 2023년에 죽었다.

📋 사건 개요
   2021년 PR #482(@minji)로 도입 — Safari 14 IndexedDB 트랜잭션
   레이스 우회용. blame은 처음에 @dohyun의 prettier 스윕을
   지목했으나, 진짜 작성자는 @minji.

🧾 증거물
   E-1  commit 4f2a91c "fix: safari indexeddb tx race" (2021, @minji)
   E-2  PR #482 — WebKit 버그 링크, 0-delay yield 설명
   E-3  BROWSERS.md — Safari 14 지원 종료 2023-06

🗣️  증인 진술
   W-1  @reviewer (2021): "추하지만 필요함 — WebKit이 고치면 제거할 것"

∴  추론
   E-2 (Safari 14 한정 버그)     ∴ 이유는 실재했다
   E-3 (Safari 14 지원 종료)     ∴ 이유는 더 이상 유효하지 않다
   ∴  Verdict: EXPIRED

⚖️  처분 권고 (결정은 당신의 몫)
   제거해도 안전. E-2의 재현 시나리오는 회귀 테스트로 남길 것.

🔗 Related   SHA: 4f2a91c   PR: #482

"이 줄은 5년간 성실히 복무했다. 명예 전역시켜 줘라."
⚖️

JUSTIFIED

이유는 실재했고 — 지금도 유효하다.

→ 건드리지 마라. 울타리엔 목적이 있다.
⌛

EXPIRED

이유는 실재했지만 이미 죽었다.

→ 지워도 안전. 주의사항 첨부.
🕳️

COLD CASE

이유를 찾을 수 없었다. alibi는 지어내지 않는다.

→ 최대한 신중히. 마지막 증인 명단 첨부.
🕊️ 사람은 절대 용의자가 아니다. 사건 파일의 모든 사람은 증인이다 — 비난의 표적이 아니라. “@minji는 용의자가 아닙니다. 이 사건의 핵심 증인입니다.”

FILE 03답할 수 있는 여섯 종류의 질문

그냥 자연스럽게 물으면 된다 — alibi가 질문의 모양을 보고 수사 모드를 스스로 고른다. 모드마다 돌아오는 보고서가 다르지만, 언제나 증거가 붙고 판결(또는 처분 권고)로 끝난다.

why-not-who기본
“이 타임아웃 왜 30초예요? 바꿔도 돼요?”
위에서 본 사건 파일: 진짜 기원 커밋, PR의 근거, 그 이유가 지금도 유효한지에 대한 판결.
commit-to-story
“커밋 4f2a91c의 전체 스토리 알려줘 — 왜 들어간 거야?”
조서: 풀려던 문제 → 리뷰에서의 논쟁 → 결정 → 배포 후 무슨 일이 있었나.
first-broken
“지난주엔 export 됐는데 지금 0을 반환해 — 깨뜨린 커밋 찾아줘.”
잠복 일지: 양 끝에서 검증한 재현 프로브, git bisect run, 범인 커밋 심문 — 사고였나 의도였나.
incident-trace
“병렬 업로드 변경, revert된 적 있어? 어느 릴리스에 탔어?”
사건 타임라인: 반입 → revert → 재반입 체인, 그리고 버그가 실린 태그·릴리스를 정확히 짚는 Exposure 라인.
design-rationale
“업로드가 왜 Promise.all이 아니라 순차 큐예요? 다른 시도는 없었어요?”
판례 기록: 설계의 시대 구분, 시도됐다가 기각된 대안들(증거 포함), 그 결정이 오늘도 유효한지.
repo-tour
“방금 합류했어 — 이 리포 역사로 온보딩해 줘. 시체 어디 묻혀 있어?”
다섯 장면 관할구역 투어: 변경 최다 핫스팟, 흉터(revert 사가), 건드리면 안 되는 고참 코드, blame 위생 — 그리고 가장 안전한 첫수.

FILE 04담당 형사

personnel file — homicide div.

강력계 형사 — 경력 20년, 검거율 0%, 의도된 기록. 신입 시절 선배의 어긋난 rebase가 그가 쓴 적 없는 핫픽스에 그의 이름을 남겼고, 추궁당한 건 그 혼자였다. 그날 이후 그는 blame이 가리키는 이름을 믿지 않는다 — “누가 이랬어” 대신 “여기서 무슨 일이 있었나”를 묻는다. 승인도 기각도 할 권한이 없다. 사실만 확정한다 — 결정은 언제나 당신의 몫이다.
